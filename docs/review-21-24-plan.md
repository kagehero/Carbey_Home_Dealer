# レビュー ㉑〜㉔ 対応

クライアントのレビューは①〜㉝まであり、順番に対応中。㉑〜㉔は「以前の画面（ダミー削除前）」に対するレビュー。

## ㉑ グラフの縦軸 ✅ 実装済み
- 折れ線グラフ（LineChart / DarkLineChart）に **Y軸の数値目盛り**を追加。売上等の水準が読める。
- `valueFormat` / `unit` で単位表記も可能。
- ※対象の「売上・利益の推移」グラフ自体は実データ源が無く削除済み。将来グラフを出す際は縦軸付きで表示される。

## ㉒ 進捗停滞の加盟店へWEBチャットでリマインド ✅ 実装済み（手動・第1段）
- 本部オンボーディング一覧で**停滞判定**（稼働中・進捗<100%・登録から7日以上）→「停滞」バッジ表示。
- 「リマインド」ボタン一押しで、その加盟店のWEBチャットに本部発のリマインドメッセージを送信。
- サマリに「停滞」件数を追加。
- 将来：一定日数停滞で**自動送信**（定期実行基盤）を第2段として検討。

## ㉔ 遷移できないボタン（説明）

- 削除済みのダミーボタン（車両追加等）は現コードには存在しない。
- 改善余地：準備中項目に「準備中」ラベルを明示（現状は丸のみ）。要望あれば対応。

## ㉓ 車両進捗管理を全紐付け（設計案・大規模新規機能）

### 目的
車両（仕入れ〜清算）の進捗を、加盟店管理・ダッシュボード・オーダーと連動して一元管理する。
車両進捗管理は現在「準備中（未実装）」。テーブルから新規に作る。

### データモデル案（migration 021 予定）
`portal.vehicles`
| カラム | 用途 |
| --- | --- |
| id | PK |
| member_id | 担当加盟店（→ members 紐付け・㉓の核） |
| order_id | 元になった仕入れオーダー（→ orders 紐付け・任意） |
| maker / car_model / year / mileage | 車両情報 |
| stage | 進捗ステージ：`sourcing`(仕入)→`prep`(商品化)→`listing`(販売中)→`sold`(清算済み) |
| purchase_price_yen / sale_price_yen / cost_yen | 仕入・販売・整備費 |
| profit_yen（算出） | 粗利 |
| purchased_at / listed_at / sold_at | 各日付 |
| note | 備考 |

### 連動（㉓の要）
- **加盟店管理**：会員個別画面に「担当車両の一覧・ステージ別件数」を表示、車両管理へ遷移。
- **ダッシュボード**：本部ダッシュボードにステージ別カンバン（仕入/商品化/販売中/清算済み）を**実データ**で表示。売上・粗利の推移も vehicles.sold から実集計 → ㉑のグラフに実データを供給できる。
- **オーダー**：オーダー完了時に車両レコードを起票（order_id 紐付け）。
- **加盟店側**：自分の車両の進捗を閲覧（半自動フローのリループと連携）。

### フェーズ分割案
1. vehicles テーブル + 型 + lib（CRUD・ステージ遷移）
2. 本部：車両進捗管理画面（カンバン）＋会員個別画面に車両サマリ
3. ダッシュボード連動（ステージ別件数・売上/粗利の実集計 → グラフ実データ化）
4. オーダー→車両起票、加盟店側の車両閲覧

### 要確認（着手前）
- ステージの定義（仕入/商品化/販売中/清算済みで確定か、他に「商談中」等を挟むか）
- 加盟店は車両を自分で登録するのか、本部が起票するのか（自動化方針との整合）
- 売上・粗利の計上タイミング（清算時 sold_at か）
