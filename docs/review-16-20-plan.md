# レビュー ⑯〜⑳ 設計案（プラン連動・自動/半自動の二系統フロー・CRM連動・完全自動化）

クライアントレビュー（画像 ⑯⑰ 付き）。前回 ⑨〜⑮ に匹敵する大規模改修のため、
まず設計を文書化し認識合わせ → 段階実装 → 各段階でDB検証、の順で進める。

---

## 0. 現状（実コードで確認済み）
- プラン型：`PlanType = 'semi_auto' | 'full_auto'`（既存）。members は `plan_id` で plans を参照。
- オンボーディング：**単一フロー**。`seed_onboarding_tasks(p_member_id)` はプランを見ずに固定10タスクを生成。
- 本部オンボーディング管理：加盟店ごとに手動でタスク状態を切替（＝⑯で「地獄」と指摘された構造）。
- CRM：`crm_customers.member_id` カラムは**存在するが**、`listCustomers` は member 情報を結合せず、画面にも加盟店の紐付けが出ていない（⑱）。
- プラン管理画面と会員管理画面：`plan_id` の割当はできるが、双方向・可視化・連動が弱い（⑰⑲）。

---

## ⑯ 完全自動化（本部の手動管理をなくす）
**要望**：加盟店ごとに本部が手動でタスクを進める構造は運用が破綻する。すべて自動で流れる形にしたい。

**方針**：
- link_key タスク（本人確認/資金/規約/マニュアル）は既に自動同期済み（フェーズ⑤）。これを徹底。
- 残る手動（manual）タスク＝「加盟契約の締結」「全項目完了の確認」を極力**自動判定**に置換：
  - 「加盟契約の締結」→ `members.contract_date` が入ったら自動 done（契約日＝締結の証跡）。
  - 「全項目完了の確認」→ 他の必須タスクが全 done なら自動 done。
  - 本人確認の「承認」だけは人の目視が要るため manual 承認を残すが、承認操作は会員詳細の1クリックで完結（既存）。
- 本部オンボーディング画面は「監視ダッシュボード」に位置づけ直す（手動で1つずつ進める運用を廃止）。

## ⑰ 自動売買 / 半自動売買 の二系統フロー ＋ プラン可視化・連動
**要望**：
- マニュアル（フローチャート）は **自動売買用** と **半自動売買用** で別スキーム。今ある全マニュアルは半自動用。
- 自動売買マニュアルも「資金調達有無」の枝分けは同じ（資金調達有の分岐も必要）。
- 加盟店個別管理で**プラン割当の可視化・反映・連動**がされていない → 全体見直し。
  - 会員個別画面に **プラン表示 / 進捗（割合）/ オーダー状況** を可視化。
  - 会員個別画面から **オーダー管理画面へ遷移**できる導線。

**方針**：
- オンボーディング/マニュアルを **プラン（semi_auto / full_auto）でフロー分岐**させる。
  - `seed_onboarding_tasks` をプラン参照に改修：full_auto なら自動売買フロー、semi_auto なら半自動フロー。
  - 実践マニュアル（manual_sections）も**フロー種別で出し分け**（section に `flow` 列を追加：`semi` / `auto` / `both`）。
  - 資金調達分岐（自己資金/資金調達）は**両フロー共通**で維持。
- 会員個別画面（/admin/members/[id]）に：プランバッジ、オンボーディング進捗バー、オーダー件数、
  「オーダー管理を見る」リンク（/admin/orders?member=…）を追加。

## ⑱ CRM と加盟店情報の連動
**要望**：CRM が加盟店情報と連動していない。
**方針**：
- `crm_customers.member_id`（既存）を活用し、CRM 顧客一覧に**担当加盟店名を結合表示**。
- CRM 顧客の新規作成/編集で**加盟店を選択・紐付け**できるようにする。
- 会員個別画面から、その加盟店に紐づく CRM 顧客/商談を参照できる導線（任意）。

## ⑲ プラン管理 ⇔ 加盟店管理 の相互連動
**要望**：プラン管理画面と加盟店管理画面が紐付いていない。どちらでも設定できるように。
**方針**：
- 会員個別画面のプラン選択（既存 plan_id）＝そのまま「加盟店側からのプラン設定」。
- プラン管理画面に「このプランの加盟店一覧」を表示し、**プラン側から加盟店へプランを割当/変更**できる導線を追加。
- どちらで変更しても members.plan_id が真実として同期（片方向の実装を双方向に）。

## ⑳ 登録→プラン選定→権限割当→フロー分岐（要件定義書準拠）
**要望**：加盟店登録 → プラン選定 → 権限割当 → フローチャート型マニュアルで自動化しながら進む構造。
- プラン選定で「自動売買モデル付帯 / 半自動を持たない」権限 → **自動売買フロー**のマニュアルを表示。
- 「自動売買も半自動も権限保有」だが半自動をやりたい加盟者 → **手動で切替**して半自動フローでスタート、
  かつ**何度もリループ**できる仕様。

**方針（プラン→権限→フローのモデル）**：
- プランに「保有モデル」を持たせる：`has_semi`（半自動可）/ `has_auto`（自動可）。
  - full_auto プラン = has_auto（+ 通常 has_semi も可）、semi_auto プラン = has_semi のみ、等をプランで定義。
- 加盟店が**実行するフロー種別**を保持：`members.active_flow`（'semi' | 'auto'）。
  - 既定は保有モデルから決定（auto のみ保有→auto、semi のみ→semi、両方→既定 auto だが切替可）。
- 両方保有の加盟者は**フロー切替UI**で semi ⇄ auto を選べ、選んだフローのマニュアルで開始・**リループ**（完了後もう一度回せる）。

## 実装フェーズ分割案（各フェーズ = migration + 画面 + 検証）
1. **プラン連動の可視化**（⑰前半・⑲）：会員個別にプラン/進捗/オーダー可視化＋オーダー画面導線、プラン⇔会員の双方向割当。
2. **プランにモデル保有フラグ**（⑳基盤）：plans に has_semi/has_auto、members に active_flow 追加。
3. **フロー二系統化**（⑰⑳）：seed をプラン/flow 分岐に、manual_sections に flow 列、加盟店フロー切替＋リループ。
4. **完全自動化**（⑯）：契約締結・全項目確認を自動判定化、本部画面を監視ダッシュボード化。
5. **CRM 連動**（⑱）：CRM 顧客に加盟店結合表示・紐付け編集。

## 確定事項（クライアント回答 2026-07-11）
- **プラン→モデル**：フルオート = has_auto **かつ** has_semi（両方保有）。セミオート = has_semi のみ。
- **自動売買マニュアルの中身**：後でクライアントが内容提供 → **器（分岐・ゲート・リループ）を先に作り、CMS で内容を埋める**。
  section を flow 種別で出し分け、本文は本部が後入力。
- **リループ**：半自動は **1仕入案ごとに繰り返す**（1台仕入れるたびにフローを回す。完了後「もう一件始める」でリセット再開）。
- **本部の手動**：**本人確認の承認のみ**。契約締結（contract_date入力で自動 done）・全項目確認（他必須が全 done で自動）は自動化。

## フロー種別モデル（確定）
- `plans.has_semi` / `plans.has_auto`（bool）でプランの保有モデルを定義。
- `members.active_flow`（'semi' | 'auto'）＝加盟店が今実行しているフロー。
  - auto のみ保有 → 'auto' 固定。semi のみ → 'semi' 固定。両方保有 → 既定 'auto'、**手動で 'semi' に切替可＆リループ**。
- オンボーディング seed とマニュアル（manual_sections.flow）を active_flow で出し分け。資金調達分岐は両フロー共通。
